## 📝 贡献流程

### 1\. Fork 仓库

将项目仓库 Fork 到你的 GitHub 账号下。

### 2\. 创建分支

从 `main` 分支新建贡献分支，命名规范：

```bash
# 新增内容/功能
feature/[模块]-[简述]
例：feature/history-high-speed-rail

# 修复问题
fix/[类型]-[简述]
例：fix/image-link-error

# 文档优化
docs/[文件]-[简述]
例：docs/readme-update
```

### 3\. 完成修改

- 内容准确、来源可靠、无侵权风险

- 图片放入 `images/` 对应目录，使用**相对路径**

- 遵循统一 Markdown 格式，不随意改动目录结构

- AI 生成内容需人工校验，标注辅助来源

### 4\. 提交 Commit

信息清晰规范，示例：

```bash
feat: 新增历史演进厅·中国高铁发展章节
fix: 修复守夜人的光轨图片路径错误
docs: 完善贡献指南与AI内容规范
ai: 辅助整理转向架技术说明并人工校对
```

### 5\. 提交 Pull Request

- 目标分支：原项目 `main`

- PR 描述需写明：修改内容、原因、依据 / 来源

- 等待维护者评审与合并

## 📋 内容规范（必看）

1. **准确性**
数据与术语权威规范，重要观点标注来源；AI 内容必须**人工审核**。

2. **可读性**
面向大众科普，通俗易懂，减少晦涩专业表述。

3. **版权合规**
文字、图片、素材自主创作或拥有合法授权，禁止侵权。

4. **格式统一**
统一标题层级、列表、代码块、图片路径规范。

5. **结构规范**
   内容放入对应目录：

    - 核心展厅 → `/核心展厅/`

    - 特展专题 → `/特展/`

    - 图片资源 → `/images/`

---

## 🤖 AI 内容贡献特别规范

- 允许使用 AI 辅助整理、润色、生成初稿

- **必须人工校对**事实、数据、原理、时间线

- 禁止直接发布 AI 未校验内容

- 建议在 Commit/PR 中注明 “AI 辅助 \+ 人工审核”
